Panduan Update Hosting

Dokumen ini berisi panduan langkah demi langkah untuk memperbarui aplikasi Koperasi SKF di hosting (cPanel) dengan aman dan mudah.

JANGAN PERNAH MENIMPA (OVERWRITE) 2 FILE INI:
1. public_html/index.php
2. kopkarskf/.env

Kedua file tersebut berisi konfigurasi kunci ("Pondasi") agar aplikasi berjalan. Jika tertimpa file dari laptop, website akan mati.

Skenario 1: Update Rutin (Kodingan) Mudah ‚Ä¢ 5 Menit

Gunakan cara ini jika Anda hanya mengubah **Fitur, Controller, Route, atau View** (tanpa menambah library baru).

  1. Persiapan di Laptop

    Pilih folder utama aplikasi Anda (biasanya di C:\laragon\www\Koperasi). Pilih folder-folder berikut lalu jadikan file .ZIP:

    • app
    • config
    • resources
    • routes
  2. Upload ke Hosting

    Buka File Manager cPanel, masuk ke folder kopkarskf.

    Klik tombol Upload -> Pilih file ZIP tadi.

  3. Ekstrak & Timpa

    Klik kanan file ZIP yang sudah terupload -> Extract.

    Saat ditanya, pastikan path tujuannya adalah folder kopkarskf.

    Biarkan proses *overwrite* selesai.

Skenario 2: Update Tampilan (Assets) Mudah

Lakukan ini jika Anda mengubah desain CSS, Javascript, atau menjalankan perintah npm run build di laptop.

  1. Zip Folder Build

    Di laptop, masuk ke folder public. Zip folder bernama build.

  2. Upload, Extract, Overwrite

    Upload ke folder public_html. Extract di sana.

Skenario 3: Major Update (Library Baru) Agak Lama

Gunakan ini HANYA jika Anda menginstal package baru via Composer.

  • Sama seperti Skenario 1, tapi IKUTKAN folder vendor ke dalam ZIP.
  • Setelah ekstract, cek Permission folder koppkarskf/vendor harus 755 dan file di dalamnya 644.

Skenario 4: Update Database (Migrasi) Penting

Karena hosting cPanel tidak menyediakan Terminal (SSH), Anda tidak bisa menjalankan php artisan migrate secara langsung. Gunakan salah satu cara ini jika Anda mengubah tabel database.

Cara Otomatis (Pakai Script Migrator)

  1. Upload Kode Database

    Pastikan Anda sudah mengupload folder database di Skenario 1.

  2. Buat Pemicu (Script)

    Buat file baru bernama migrator.php di dalam folder public_html. Isi dengan kode ini:

    <?php error_reporting(E_ALL); ini_set('display_errors', 1); // Load Laravel require __DIR__ . '/../kopkarskf/vendor/autoload.php'; $app = require_once __DIR__ . '/../kopkarskf/bootstrap/app.php'; echo "<h2>Menjalankan Migrasi Database...</h2>"; try { $kernel = $app->make(Illuminate\Contracts\Console\Kernel::class); $kernel->call('migrate', ['--force' => true]); echo "<pre>" . $kernel->output() . "</pre>"; echo "<h3 style='color:green'>SUKSES! Database Terupdate.</h3>"; } catch (Exception $e) { echo "<h3 style='color:red'>GAGAL:</h3>"; echo $e->getMessage(); } ?>
  3. Jalankan

    Buka browser: kopkarskf.com/migrator.php.

    PENTING: Segera hapus file migrator.php setelah selesai dipakai.

Aplikasi Error setelah Update?

Langkah pertolongan pertama (P3K) paling ampuh adalah Membersihkan Cache.

  1. Buka File Manager -> kopkarskf/bootstrap/cache/.
  2. Hapus semua file PHP di dalamnya (kecuali .gitignore).
  3. Refresh website.
